SEVERE FIXED Issue #2: Broken Error Handling with next

Location: preamble.R line 188

Problem: The next statement is used outside of any loop. In R, this causes undefined behavior - either an error or silent continuation with undefined variables.

# BEFORE
tryCatch({
  beta<-solve(...)
  ...
}, error = function(e) { skip_to_next <- TRUE})

if(skip_to_next) { next }   # BUG: 'next' outside loop!

# AFTER
tryCatch({
  beta<-solve(...)
  ...
}, error = function(e) {
  stop("Analytic estimator failed. Check for collinearity in instruments.")
})

Rationale: If the analytic IV estimator fails (e.g., due to singular matrices), the code should stop with a clear error message rather than silently continuing with undefined beta and beta_2 variables, which would cause cryptic errors later.

MODERATE FIXED Issue #3: No Convergence Checking After Optimization

Location: 02_00_estimation_inversion.R lines 39-42, 89-91

Problem: Results from BBsolve and optim are used without checking if optimization converged.

# BEFORE

Rationale: Using non-converged optimization results produces garbage estimates. Added warnings (not errors) to alert the user while still allowing inspection of partial results.

MODERATE FIXED Issue #4: Inefficient Repeated Matrix Inversion

Location: preamble.R lines 179-183

Problem: solve(t(z_mm_1)%*%z_mm_1) is computed twice in the same expression.

# BEFORE
beta<-solve(t(mm_1)%*%z_mm_1%*%solve(t(z_mm_1)%*%z_mm_1)%*%t(z_mm_1)%*%mm_1)%*%
      (t(mm_1)%*%z_mm_1%*%solve(t(z_mm_1)%*%z_mm_1)%*%t(z_mm_1)%*%as.matrix(estim_matrix[,"log_rel_mkt"]))

# AFTER
ZtZ_inv <- solve(t(z_mm_1) %*% z_mm_1)
Zt <- t(z_mm_1)
proj_z <- z_mm_1 %*% ZtZ_inv %*% Zt  # Projection matrix
beta <- solve(t(mm_1) %*% proj_z %*% mm_1) %*%
        (t(mm_1) %*% proj_z %*% as.matrix(estim_matrix[,"log_rel_mkt"]))

Rationale: Matrix inversion is O(n³). Precomputing saves one full inversion per execution. With large matrices, this is a measurable performance improvement.

MODERATE FIXED Issue #5: Missing Cluster Cleanup

Location: 02_00_estimation_inversion.R line 32-35

Problem: Parallel cluster is created but never explicitly stopped, causing potential resource leaks.

# BEFORE
if (get_os()=="windows"){
  clust <- makeCluster(core_count)
  clusterExport(clust, ls())
  clusterEvalQ(clust, library("SQUAREM"))
}
# ... code runs ...
# cluster never stopped!

# AFTER
if (get_os()=="windows"){
  clust <- makeCluster(core_count)
  on.exit(stopCluster(clust), add = TRUE)  # Ensure cleanup
  clusterExport(clust, ls())
  clusterEvalQ(clust, library("SQUAREM"))
}

Rationale: on.exit() ensures the cluster is stopped even if the script errors out, preventing orphaned R processes from consuming system resources.

MODERATE FIXED Issue #6: Inconsistent Tolerance Levels

Location: 02_00_estimation_inversion.R lines 16-18

Problem: Inner tolerance (1e-08) is much tighter than outer tolerance (1e-02), wasting computation on precision that doesn't affect final results.

# BEFORE
innertol<-1e-08
outertol<-1e-04
obj_tol<-1e-02

# AFTER
innertol<-1e-06   # Relaxed: no point being more precise than outer loop
outertol<-1e-04   # Unchanged
obj_tol<-1e-04    # Tightened: was too loose for reliable convergence

Rationale: The inner fixed-point iteration was solving to 1e-08 precision, but the outer bisection only needed 1e-04 precision, and the final objective only checked 1e-02. This wastes computation. Rebalanced tolerances for efficiency while tightening the final objective tolerance for more reliable results.
